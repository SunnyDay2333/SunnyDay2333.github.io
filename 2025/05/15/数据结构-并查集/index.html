并查集

并查集的作用：

1、将两个集合合并。

2、询问两个元素是否在一个集合中。

这两个操作的时间复杂度近似于O(1).

基本原理

每个集合用一个树来表示，树根的编号就是集合的编号，每个节点用p[x]来存储元素x的父节点，只有树根的p[x]=x，通俗理解就是集合里每个人都只需要记得自己的上级是谁，判断是否在集合里就一直向上查找，直到找到了树根，树根的编号就是集合的编号，那么这个元素就在这个集合里。

合并就是，对于两个元素x,y，先判断是否属于一个集合，都开始向上查找，找到它们对应的根节点，如果根节点不同，不属于一个集合，把任意一个根节点作为另一个根节点的子节点即可，这样两个集合里所有元素都有了共同的根节点，也就属于一个集合。

路径压缩（很重要，代码极其精巧）

这是如何实现的呢？

先递归再赋值，如果不是根节点，就递归，查找p[x]的父节点，查到根节点之后返回，递归返回值，每个节点的p[x]都指向了根节点，仅仅两行代码就实现了路径压缩，优雅。

调用示例：

假设父节点是d： a → b → c → d，执行find(d)：

1、find(d): p[d] = c != d , p[d]=c =find(p[d])=find(c);

2、find(c): p[c] = b != c , p[c] =b = find(p[c]) = find(b);

3、find(b): p[b] = a != b , p[b] = a =find(p[b]) = find(a);

4、find(a): p[a]==a, return a;

现在开始返回

3、p[b] =find(a) = a return a;

2、p[c] = find(b)= a return a;

1、 p[d] = find(c)= a return a;

现在不仅查到了d属于哪个集合，而且d的所有父节点都直接连到了根节点，以后如果需要查这条路径的元素，复杂度就是O(1)
